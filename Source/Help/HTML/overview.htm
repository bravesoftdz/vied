Overview

The Version Information Editor greatly simplifies the creation of VERSIONINFO resource statements for inclusion in Windows resource files. These resources, when compiled by a resource file compiler, enable version information to be embedded in application, DLL and other types of file recognised by Windows.

Key features of the editor are that it:

  • Provides a graphical way of editing all the different components of a version information resource and can validate changes.
    Learn more.
  • Has special macro-like fields can be embedded in string file information that automate some common update tasks, such as keeping the version numbers in the ProductVersion and FileVersion string info items in sync with those in fixed file info.
    Learn how to use fields.
  • Can check edited documents for errors.
    Learn more.
  • Can be configured to work with a 3rd party resource compiler to generate binary resource files.
  • Supports comments in both the native version information file format and in exported resource files.
  • Uses its own text based file format to store version information for later updating. The file format is simple enough to be hand edited if necessary.
  • Can be used in automated build systems to generate resource source files from native version information files. This is achieved by using one of the progam's command line options that cause it to perform the code generation silently, without displaying a window.

For further information on using the program see the How Do I? topics.
