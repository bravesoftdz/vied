Version Information Expert

Bugs and Improvements

This documents list all reported bugs for Version Information Expert, along suggestions for future improvements and updates. All implemented improvements and bugs fixed are also listed.

The main sections are:

  • Bug Reports
  • Update Suggestions

Bug Reports

  • None outstanding.

Bug Fixes

  • 32bit version of program is treating ProductVersion resource string as not being permitted whenever supplied and, quite rightly, treating it as a required string when not present!
    • Fixed v2.1

  • Indentation in comments is being lost when a VI file is re-loaded.
    • Fixed v2.2

  • Indentation of comments is being increased by one additional space.
    • Fixed v2.3

  • When no resource compiler has been specified the program always prompts the user to enter details at start-up, regardless of whether user has checked the box requesting that the prompt not be shown
    • Fixed v2.9.1

  • If no file extension is specified when saving .vi files the program does not append one -- this is non-standard behaviour.
    • Fixed v2.10.1

  • The BRCC32 compiler was failing when passed file names containing spaces.
    • Fixed v2.10.1 by (a) making the default parameter string for BRCC32 wrap the command line parameters in quotes and (b) adding a note to the help file noting that users should do the same when manually adding command lines

Update Suggestions

  • Allow program to read in resource info from an existing executable - use TPJVersionInfo component - this provides nice way to update info when writing a new version of program.

  • Allow version information to come from existing resource binary (*.res) file.

  • Allow version information to come from existing resource source (*.rc) file or from version information resource statement pasted into application -- difficult.

  • Give option for more than one preferred default settings for new vi file.

  • Allow user to delete the preferences (restore defaults).

  • Allow an item to be "deleted" when it is highlighted and Del key is pressed (and add Edit | Delete menu option to do same. What "delete" means must depend on context - fixed file info items could be zeroed, strings cleared, language and char-set reset to defaults (preferences?).

  • Allow user to configure how file version and product version numbers are displayed. (eg: Number, Upper Case Letter, Lower Case Letter).

  • Provide menu option to allow file and product version number to be incremented (need dialogue to say whether file and/or product info should be updated, need to specify which of 4 major/minor version numbers to update, need to say whether to increment by 1 or to some given value. Need to be able to record prefered options.

  • Make the comments editor dialog box resizeable (with persistent size).

  • Make comments editor dialog box display a grey vertical line at 80 char margin so user knows when to wrap round comments so they display in Delphi editor OK.

  • Descend the two sorts of string editor dialogs from a common root class rather than switching states using properties of single class.

  • Move some of dlg box "construct, use, destruct" code that is in main form into class procedures and functions of the dialog box classes.

  • Add Contents dialog to help system.
